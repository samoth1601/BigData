For example, businesses and organizations can do product and service benchmarking by finding customer sentiments and opinions without getting explicit feedback. Also, it can be used to place ads in user-generated content or for providing a general search for opinions. </p><p></p><p>One major key challenge to computer science is to stitch the world’s structured information and knowledge together.  So that it can give a conceptual understanding of the world to computers. This field of study is known as the semantic web. A major component of the semantic web is called ontologies. Using an ontology query language, we can query an ontology and get meaningful information from web pages. The semantic web brings structure to the meaningful content of the web pages, so that software agents visiting a web page will know more about the web page rather than just keywords. </p><p></p><p>In this report we will present a web application that uses a dataset consisting of microblog posts and news articles to do sentimental analysis. The mood is be determined for a given timespan, determined by the user in the way of two variables, start, and end-date. The assumption of mood is be based on Erna’s tweets from her Twitter feed. The result, her mood, will be shown on a web page.</p><p><br>We are also fetching news articles from the same time period relevant to Erna and Norway as a nation, from the norwegian newspaper VG’s web pages. We then do sentiment analysis on the articles, and display them on our web page, the hope being they relate to Erna’s mood. In addition, we use SPARQL queries on Erna with the DBpedia ontology. We use these to show facts about Erna, these may be relevant to her current mood, but are mostly for fun. </p><h3 id="user-content-h.au4k72h0syqw"><a id="user-content-motivation" class="anchor" href="#motivation" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Motivation</h3><p>We wanted to analyze the mood of Erna Solberg, the prime minister of Norway’s, tweets and see this in relation to news about Norway and the prime minister. We use semantics and sentimental analysis.</p><h4 id="user-content-h.5905qcy12aaj"><a id="user-content-semantics" class="anchor" href="#semantics" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Semantics</h4><p>The semantic web brings structure to the content of web pages, so that it can produce semantic meaning. </p><h5 id="user-content-h.957id9m0f0hj"><a id="user-content-web-ontologies" class="anchor" href="#web-ontologies" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Web ontologies</h5><p>An ontology formally describes a domain of discourse. The ontologies purpose is to provide a shared understanding of the domain of a web page. It usually consists of a finite number of terms and relationship between the terms. The concepts (classes) of the domain is denoted by these terms. For example, in the domain of politics,  prime minister, election, and constitution are some important concepts. The relationships is hierarchies of classes. Furthermore, ontologies may include information about properties, value restrictions, disjointness statements, and logical relationships between objects.</p><p></p><h5 id="user-content-h.kra6xm2l29hm"><a id="user-content-rdf" class="anchor" href="#rdf" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>RDF</h5><p>Ontologies can be constructed by using an ontology language. An important ontology language is called RDF (Resource Description Framework). RDF is a data model for resources, where a resource is a thing in the world that someone want to know more about. Every resource is identified by a URI (Universal Resource Identifier), which can be a web address or some other unique identifier.  A special kind of resources is called properties. Properties describe relations between resources (“written by”, “age”) and is also identified by using URIs. The last fundamental concept of RDF is called statements, which assert the property of resources. Statements is an object-attribute-value triple that consists of a resource, a property, and a value. The total information in a RDF is distributed triples.</p><p></p><h5 id="user-content-h.3ewckcrkh3q9"><a id="user-content-sparql" class="anchor" href="#sparql" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>SPARQL</h5><p>For getting information from RDF data one can use an ontology query language called SPARQL (SPARQL Protocol and RDF Query Language). It provides the possibility to extract information in the form of URIs, blank nodes, plain, and typed literals. It can be used to extract RDF subgraphs or constructing new RDF graphs.</p><p></p><p>In this project we use DBpedia, which provides a RDF for wikipedia.org. We extract information from this RDF by using SPARQL.</p><p></p><h4 id="user-content-h.mqs3v912gup3"><a id="user-content-sentiment-analysis" class="anchor" href="#sentiment-analysis" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Sentiment analysis</h4><p>Sentimental analysis focuses on analyzing text consisting of opinions, target of opinions and the opinion holder. An opinion is a quintuple (ej, ajk, soijkl, hi, tl,), where ej is the target entity, ajk is the aspect of the entity ej, soijkl is the sentiment value, hi is the opinion holder, and tl is time when the opinion is expressed. </p><p></p><h5 id="user-content-h.yl76xboczk9q"><a id="user-content-document-sentiment-classification" class="anchor" href="#document-sentiment-classification" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Document Sentiment Classification</h5><p>In document sentiment classification, the task is to classify if the whole document express a positive or a negative sentiment. To do this one have to find the opinion for a document d evaluating a general entity. The opinion can then be expressed as the quintuple, (_, GENERAL, s, _, _), where we try to determine the overall sentiment s.</p><p>One technique for doing document sentiment classification is an unsupervised approach called the lexicon-based method.</p><p></p><h5 id="user-content-h.bsswjt916etc"><a id="user-content-unsupervised-lexicon-based-approach" class="anchor" href="#unsupervised-lexicon-based-approach" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Unsupervised lexicon-based approach</h5><p>The unsupervised lexicon-based approach can be done with a bag of word approach and then averaging the word polarities. [ARTICLE]<sup><a href="#cmnt1" id="user-content-cmnt_ref1">[a]</a></sup> suggest preprocessing the words in the following way:</p><p></p><p>1. Tokenization</p><p>2. POS-tagging</p><p>3. reduce text to nouns, adjectives, verbs, adverbs</p><p>4. normalization: stemming and/or lemmatization</p><p></p><p>Then the word polarities are calculated by matching the words with a lexicon containing a sentiment for each word. Once the scores have been retrieved, we can combine the score of the whole text by averaging all the scores.</p><h5 id="user-content-h.5sea9ayef3ti"><a id="user-content-multilingual-sentiment-analysis" class="anchor" href="#multilingual-sentiment-analysis" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Multilingual Sentiment Analysis</h5><p>Doing sentimental analysis on text other than english can be challenging and is not explored very much.
